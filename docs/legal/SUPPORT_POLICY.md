# Support Policy

Effective date: 2026-03-23

This policy describes how support is provided for `Note to Self`.

## Support Channels

- GitHub issues: https://github.com/seanebones-lang/Note-to-Sef/issues

## What Is Covered

- Reproducible bugs and crashes.
- Data loss issues caused by app defects.
- Installation and update problems.
- Accessibility issues and major usability blockers.

## What Is Not Guaranteed

- Immediate response times.
- Custom feature implementation on request.
- Support for modified builds not distributed by project maintainers.
- Recovery of data deleted outside app-controlled storage.

## Best Effort Targets

- Critical crash/data integrity issues: initial response target 2 business days.
- General bug reports: initial response target 5 business days.

Response times are best effort and may vary.

## Required Information for Bug Reports

Please include:

- Device and OS version.
- App version/build number.
- Clear reproduction steps.
- Screenshots or logs (without sensitive personal note content).

## Security Reports

If you discover a security issue, report it privately via a GitHub issue marked as security-sensitive, and avoid public disclosure of exploit details until mitigation is available.
